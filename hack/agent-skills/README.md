# Karmada Agent Skills

A repository of structured "skills" that teach AI coding agents how to work
with Karmada — generating policies, auditing them, explaining placement, and
debugging propagation across multi-cluster Kubernetes deployments.

> **Status:** Working slice for LFX Mentorship CNCF Term 02 (2026), tracking
> [PR #7466](https://github.com/karmada-io/karmada/pull/7466). Five of seven
> planned skills are implemented end-to-end with a deterministic generator,
> a lint-style validator, regenerable worked examples, unit tests, and an
> integration shell test.

## Why agent skills?

AI agents (Claude Code, Cursor, etc.) regularly produce broken
`PropagationPolicy` and `OverridePolicy` YAML because the schema, semantics,
and idioms are not in their training data with enough density. A skill is a
small, self-contained package — `SKILL.md` + knowledge + templates +
deterministic helper scripts — that an agent loads on demand to do a
specific task correctly.

## What makes this stronger than placeholder skills

1. **The validator is real, not aspirational.** `validate-policy.py` has
   ~20 rules with stable IDs (`KMD001`, `KMD221`, …). Each rule has a
   pinned unit test, so renaming or dropping a rule is a CI failure.
2. **The generator and validator are coupled.** Every example is
   regenerated by the generator and lint-clean against the validator.
   The integration test (`tests/test-skills.sh`) re-runs the loop on
   each invocation.
3. **SKILL.md files are imperative, not advisory.** They contain
   decision tables, hard rules ("do not draft policy YAML
   token-by-token"), and explicit anti-patterns — not generic "consider
   the edge cases" guidance.
4. **Pitfalls are encoded as data.** `knowledge/policy-apis.yaml`
   carries a structured `pitfalls:` block; each pitfall has a
   corresponding validator rule and unit test. Documentation, code, and
   tests cannot drift apart silently.
5. **Caught a real bug while building.** The Karmada API field is
   `imageOverrider` (with -r), not `imageOverride`. That typo is now a
   first-class validator rule (`KMD210`) tested end-to-end.

## Running everything

```bash
# Generate a regional PropagationPolicy
python3 scripts/policy-generator.py \
  --kind PropagationPolicy --name nginx-eu --namespace default \
  --target-kind Deployment --target-name nginx \
  --clusters member-de,member-fr --replica-scheduling Divided

# Lint a policy
python3 scripts/validate-policy.py path/to/policy.yaml

# Unit + integration tests
python3 -m unittest discover -s tests -p 'test_*.py' -v
bash tests/test-skills.sh
```
